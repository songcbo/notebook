# Knowledge 目录说明

## 目录定位
- 这个目录只放“已处理、可复用”的知识。
- 分类分两层：`Work`（工作相关）和 `General`（非工作领域）。

## 怎么写
1. 每篇知识笔记至少包含：来源、核心观点、你的理解、可复用结论。
2. 工作知识放 `Knowledge/Work`，其他主题放 `Knowledge/General`。
3. 不要原文堆砌，必须写“你的结论”。
4. 尽量回链到相关项目，形成知识到执行的闭环。

## 流转关系
- 输入：来自 `Inbox/Processed` 的已处理条目，或项目复盘提炼。
- 输出：被 `Planning` 引用为行动建议，被 `Projects` 引用为实现依据。

## 常见误区
- 只有摘抄没有结论，后续难复用。
- 分类过细，导致写一篇笔记要先做复杂决策。

## 字段说明（中文注释）
- `type`: 笔记类型，知识页用 `knowledge`。
- `domain`: 领域分类，建议 `work` 或 `general`。
- `source`: 来源链接或书名。
- `status`: 推荐 `seed`（初稿）或 `solid`（已沉淀）。
- `tags`: 用于主题聚合。
