# Planning 目录说明

## 目录定位
- 这个目录只用于时间调度与执行安排：`Daily` 和 `Weekly`。
- 这里只写“要做什么、何时做、推进到哪里”，不沉淀长文资料。

## 怎么写
1. 每天新建一篇 `Daily/YYYY-MM-DD.md`，记录当天 3 个最重要任务。
2. 每周新建一篇 `Weekly/YYYY-Www.md`，定义本周目标和项目推进重点。
3. 每条任务尽量链接到具体项目或知识页，例如 `[[Projects/Agent 平台重构]]`。
4. 任务使用 Tasks 语法，方便后续聚合。

## 流转关系
- 输入：来自 `Inbox/Processed` 的已处理条目、上周未完成任务、项目状态更新。
- 输出：指向 `Projects` 的推进动作，必要时沉淀到 `Knowledge`。

## 常见误区
- 把会议纪要和资料摘录写到 Planning（应放 Projects 或 Knowledge）。
- 只列任务不链接项目，导致后续不可追踪。

## 字段说明（中文注释）
- `type`: 笔记类型。`daily` 表示日计划，`weekly` 表示周计划。
- `date`: 对应日期（如 `2026-03-03`）。
- `week`: 周编号（如 `2026-W10`）。
- `status`: 当前状态，常用 `active`。
- `tags`: 标签，建议包含 `planning`。
- Tasks 语法示例：`- [ ] 任务内容 📅 2026-03-05 🔺`。
  - `- [ ]`: 未完成任务。
  - `📅`: 截止日期。
  - `🔺`: 高优先级。
